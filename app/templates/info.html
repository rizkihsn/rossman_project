Rossmann Store Sales

Sumber: Kaggle

Deskripsi: {{ info.dataset.description }}

Total Records: 1,017,209 baris

Periode: 2013 – 2015

Lisensi: Database Contents License (DbCL) / Kaggle Competition Rules

ℹ️ Kenapa test.csv tidak ada?

Struktur kolom test.csv sama persis dengan train.csv, tetapi tanpa kolom target (Sales) dan Customers. Di dalam project ini, input manual dari *user* di halaman "Predict" menggantikan peran dari test.csv tersebut.

🎯 Target Variable

Variable: Sales

Type: Continuous (Regression)

Deskripsi: Penjualan harian per toko dalam EUR

Problem: Memprediksi jumlah penjualan harian berdasarkan faktor temporal, promosi, dan informasi toko

📝 Kolom train.csv:
  • Store: ID toko unik (1-1115)
  • DayOfWeek: Hari dalam seminggu (1-7)
  • Date: Tanggal transaksi
  • Sales: Penjualan harian (TARGET) ⭐
  • Customers: Jumlah pelanggan
  • Open: Status toko (0/1)
  • Promo: Ada promosi (0/1)
  • StateHoliday: Libur negara (0/a/b/c)
  • SchoolHoliday: Libur sekolah (0/1)
📝 Kolom store.csv:
  • StoreType: Tipe toko (a/b/c/d)
  • Assortment: Level produk (a/b/c)
  • CompetitionDistance: Jarak kompetitor (m)
  • CompetitionOpenSince: Kapan kompetitor buka
  • Promo2: Promosi lanjutan (0/1)
  • Promo2Since: Mulai promo lanjutan
  • PromoInterval: Interval bulan promo

Preprocessing Steps:
  1. ✅ Load dataset: train.csv, test.csv, store.csv
  2. ✅ Gabungkan: train + store dan test + store (left join)
  3. ✅ Handle missing values (CompetitionDistance → median, sisanya → 0)
  4. ✅ Filter baris Sales = 0 (toko tutup)
  5. ✅ Feature engineering dari Date → Year, Month, Day, Quarter, WeekOfYear
  6. ✅ Encoding: StateHoliday, StoreType, Assortment → numerik
  7. ✅ Drop kolom: Date, Customers, PromoInterval
  8. ✅ Normalisasi: StandardScaler
  9. ✅ Split: 80% train, 20% test
